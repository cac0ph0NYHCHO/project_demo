# STM32F103 无阻塞多任务智能控制系统
基于 STM32F103C8T6 的模块化、非阻塞、状态机架构综合项目  
[点击观看视频演示](https://github.com/cac0ph0NYHCHO/project_demo/blob/main/视频演示.mp4)

## 项目简介
- 本项目基于 STM32 标准库开发，实现**无阻塞多任务嵌入式系统**，包含按键输入、红外中断、CAN 总线通信、OLED 显示、PWM 呼吸灯、LED 状态机、串口日志输出等功能。  
- 全程不使用 delay 阻塞，采用**定时器时间戳 + 状态机架构**，系统响应快、稳定性高、代码结构清晰，符合企业级嵌入式开发规范。

## 硬件平台
- 主控芯片：STM32F103C8T6
- 外设模块：
  - 独立按键模块（用于触发 CAN 数据发送）
  - 红外避障传感器（EXTI 外部中断触发紧急告警）
  - OLED 显示屏（I2C 通信，实时显示 CAN 通信数据）
  - CAN 收发器（实现双板 CAN 总线数据交互）
  - LED 灯（蓝色 PWM 呼吸灯 + 红色状态指示）

## 实现功能
✅ 非阻塞系统架构（全程无 delay，避免系统卡顿）  
✅ 蓝色 PWM 呼吸灯（0-100 亮度平滑切换，非阻塞实现）  
✅ 红色 LED 状态机（3种模式：熄灭 / 慢闪-正常发送 / 快闪-紧急告警）  
✅ 按键消抖处理（无阻塞扫描，触发 CAN 数据发送并切换 LED 状态）  
✅ 红外传感器 EXTI 外部中断（触发紧急模式，中断安全设计）  
✅ CAN 总线收发功能（标准 ID 通信，支持数据双向交互）  
✅ OLED 实时显示（CAN 发送/接收 ID、数据长度、数据内容）  
✅ USART 串口日志输出（printf 重定向，支持电脑本地保存日志）  
✅ 超时自动恢复（LED 状态闪烁后，600ms 自动恢复待机状态）

## 系统架构
- 模块化驱动设计：按功能拆分独立驱动文件，降低代码耦合度
- 状态机管理：通过枚举定义 LED 工作模式，规范状态切换逻辑
- 中断安全设计：中断服务程序仅置标志位，不执行耗时操作（如 printf）
- 主循环调度：轮询调用各任务函数，保证系统实时响应，无阻塞风险

## 技术要点
- 核心框架：STM32 标准库开发
- 外设驱动：GPIO 配置、通用定时器、PWM 脉冲宽度调制、EXTI 外部中断
- 通信协议：CAN 总线通信、USART 串口通信（printf 重定向）
- 编程思想：非阻塞编程（定时器时间戳实现）、状态机设计模式
- 代码规范：模块化、低耦合、高可维护代码，注释清晰，命名规范

## 项目亮点
1. 全程无阻塞架构，基于定时器时间戳实现任务调度，系统响应时间 ≤ 10ms；
2. 采用状态机管理 LED 多模式，逻辑清晰，避免状态错乱，提升系统鲁棒性；
3. 中断安全优化，中断仅置标志位，主循环统一处理日志，规避中断阻塞风险；
4. 代码高度模块化，各功能拆分独立驱动，便于后续扩展与团队维护；
5. 集成工业级调试日志，支持电脑本地保存，便于问题追溯与项目调试；

## 使用说明
1. 开发环境：Keil MDK 5，编译时需开启 Use MicroLib（解决 printf 卡死问题）；
2. 串口配置：波特率 9600，串口日志可保存至电脑 TXT 文件，便于调试；
3. 功能操作：
   - 按键 1 触发 CAN 数据发送，红色 LED 慢闪，数据自动递增；
   - 遮挡红外传感器，触发紧急告警模式，红色 LED 快闪，串口输出紧急日志；
   - OLED 实时显示 CAN 发送 ID、接收 ID、数据长度及具体数据内容；
   - 待机状态下，蓝色 LED 呈现平滑呼吸灯效果。
